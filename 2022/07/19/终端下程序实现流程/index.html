终端下程序实现流程
Jul 19 2022

Catkin是ROS官方指定系统，用于创建功能和编译功能包，Catkin扩展了CMake，将 cmake 与 make 指令做了一个封装从而完成整个编译过程的工具。可移植性、对交叉编译更加友好。

创建工作空间并初始化

mkdir -p 自定义空间名称/src
cd 自定义空间名称
catkin_make

创建了build和devel目录:
build:编译空间，用于存放CMake和catkin的缓存信息、配置信息和其他中间文件。
devel:开发空间，用于存放编译后生成的目标文件，包括头文件、动态&静态链接库、可执行文件等。

进入src创建ROS包并添加依赖

cd src
catkin_create_pkg 自定义功能包名 roscpp rospy std_msgs

在工作空间下创建一个功能包（ROS基本单元，包含多个节点）

进入ROS包的src目录编辑源文件

cd 自定义功能包/src
touch main.cpp
sudo gedit main.cpp

#include "ros/ros.h"

int main(int argc, char *argv[])
{
    //执行 ros 节点初始化
    ros::init(argc,argv,"hello");
    //创建 ros 节点句柄(非必须)
    ros::NodeHandle n;
    //控制台输出 hello world
    ROS_INFO("hello world!");

    return 0;
}

编辑ROS包目录下的CMakelist.txt文件

sudo gedit ~/demo_ws/src/demo1/CMakeLists.txt

add_executable(任意节点名称
  src/源代码文件名.cpp
)
target_link_libraries(任意节点名称
  ${catkin_LIBRARIES}
)

进入工作空间目录、编译

cd 自定义空间名称
catkin_make

执行

  1. 启动终端1：
roscore
  1. 启动终端2：
cd 工作空间
source ./devel/setup.bash
rosrun 包名 节点名
  1. 创建工作空间并初始化
  2. 进入src创建ROS包并添加依赖
  3. 进入ROS包的src目录编辑源文件
  4. 编辑ROS包目录下的CMakelist.txt文件
  5. 进入工作空间目录、编译
  6. 执行
