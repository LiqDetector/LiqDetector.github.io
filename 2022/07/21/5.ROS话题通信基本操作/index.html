话题通信基本操作
Jul 21 2022

理论模型

1.1 角色

  • ROS Master：根据话题建立发布者与订阅者之间的连接
  • Talker：发布消息
  • Listener：订阅消息

1.2 流程

  1. Talker注册：利用RPC在ROS Master中注册自身信息（话题、RPC地址）
  2. Listener注册：利用RPC在ROS Master中注册自身信息（话题）
  3. ROS Master信息匹配：根据注册表匹配两者，利用RPC向Listener发送Talker的RPC地址
  4. Listern向Talker发送请求：Listener 利用 RPC 向 Talker 发送连接请求，传输话题、消息类型和通信协议(TCP/UDP)
  5. Talker确认请求：Talker利用RPC向Listener确认连接信息，发送TCP地址

前5步使用RPC协议进行通信

  1. 建立连接：Listener利用TCP与Talker建立网络连接
  2. Talker向Listener发送消息

建立连接后关闭ROS Master

基于C++的话题通信

2.1 场景

  • 发布方以10hz频率发布文本消息
  • 订阅方订阅消息并将消息内容打印输出

2.2 实现流程

  1. 编写发布方实现
  2. 编写订阅方实现
  3. 编辑配置文件
  4. 编译并执行

2.3 实现代码

发布方：

/*
    需求: 实现基本的话题通信，一方发布数据，一方接收数据，
         实现的关键点:
         1.发送方
         2.接收方
         3.数据(此处为普通文本)

         PS: 二者需要设置相同的话题


    消息发布方:
        循环发布信息:HelloWorld 后缀数字编号

    实现流程:
        1.包含头文件 
        2.初始化 ROS 节点:命名(唯一)
        3.实例化 ROS 句柄
        4.实例化 发布者 对象
        5.组织被发布的数据，并编写逻辑发布数据
*/
// 1.包含头文件 
#include "ros/ros.h"
#include "std_msgs/String.h" //普通文本类型的消息
#include <sstream>

int main(int argc, char  *argv[])
{   
    //设置编码
    setlocale(LC_ALL,"");

    //2.初始化 ROS 节点:命名(唯一)
    // 参数1和参数2 后期为节点传值会使用
    // 参数3 是节点名称，是一个标识符，需要保证运行后，在 ROS 网络拓扑中唯一
    ros::init(argc,argv,"talker");
    //3.实例化 ROS 句柄
    ros::NodeHandle nh;//该类封装了 ROS 中的一些常用功能

    //4.实例化 发布者 对象
    //泛型: 发布的消息类型
    //参数1: 要发布到的话题
    //参数2: 队列中最大保存的消息数，超出此阀值时，先进的先销毁(时间早的先销毁)
    ros::Publisher pub = nh.advertise<std_msgs::String>("chatter",10);

    //5.组织被发布的数据，并编写逻辑发布数据
    //数据(动态组织)
    std_msgs::String msg;
    // msg.data = "你好啊！！！";
    std::string msg_front = "Hello 你好！"; //消息前缀
    int count = 0; //消息计数器

    //逻辑(一秒10次)
    ros::Rate r(1);

    //节点不死
    while (ros::ok())
    {
        //使用 stringstream 拼接字符串与编号
        std::stringstream ss;
        ss << msg_front << count;
        msg.data = ss.str();
        //发布消息
        pub.publish(msg);
        //加入调试，打印发送的消息
        ROS_INFO("发送的消息:%s",msg.data.c_str());

        //根据前面制定的发送贫频率自动休眠 休眠时间 = 1/频率；
        r.sleep();
        count++;//循环结束前，让 count 自增
        //暂无应用
        ros::spinOnce();
    }


    return 0;
}

订阅方：

/*
    需求: 实现基本的话题通信，一方发布数据，一方接收数据，
         实现的关键点:
         1.发送方
         2.接收方
         3.数据(此处为普通文本)


    消息订阅方:
        订阅话题并打印接收到的消息

    实现流程:
        1.包含头文件 
        2.初始化 ROS 节点:命名(唯一)
        3.实例化 ROS 句柄
        4.实例化 订阅者 对象
        5.处理订阅的消息(回调函数)
        6.设置循环调用回调函数

*/
// 1.包含头文件 
#include "ros/ros.h"
#include "std_msgs/String.h"

void doMsg(const std_msgs::String::ConstPtr& msg_p){
    ROS_INFO("我听见:%s",msg_p->data.c_str());
    // ROS_INFO("我听见:%s",(*msg_p).data.c_str());
}
int main(int argc, char  *argv[])
{
    setlocale(LC_ALL,"");
    //2.初始化 ROS 节点:命名(唯一)
    ros::init(argc,argv,"listener");
    //3.实例化 ROS 句柄
    ros::NodeHandle nh;

    //4.实例化 订阅者 对象
    ros::Subscriber sub = nh.subscribe<std_msgs::String>("chatter",10,doMsg);
    //5.处理订阅的消息(回调函数)

    //     6.设置循环调用回调函数
    ros::spin();//循环读取接收的数据，并调用回调函数处理

    return 0;
}

mkdir -p demo_topic/src
cd demo_topic/
catkin_make
  1. 创建功能包并添加依赖
cd src
catkin_create_pkg topic_cpp roscpp rospy std_msgs
  1. 进入topic_cpp功能包src目录编辑源文件（如上节代码）
cd topic_cpp/src/
touch Hello_pub.cpp
touch Hello_sub.cpp

sudo vim Hello_pub.cpp
sudo vim Hello_sub.cpp
  1. 编辑topic_cpp目录下CMakeLists.txt文件（如上节代码）
cd ..
sudo vim CMakeLists.txt
  1. 进入demo_topic工作空间并编译
cd ~/demo_topic/
catkin_make
  1. 执行
    roscore
cd demo_topic
source ./devel/setup.bash
rosrun topic_cpp Hello_pub

cd demo_topic
source ./devel/setup.bash
rosrun topic_cpp Hello_sub
